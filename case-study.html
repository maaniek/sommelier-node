How this was actually built

Not one tool. The right tool, every time — that's the skill.

Advanced file architecture CLAUDE.md + structured project folders — every session picked up exactly where the last one left off
Skills, not raw prompts brainstorming → real requirements pass. pptx → native editable deck
Right tool per asset Product renders generated in Gemini (Nano Banana), composited straight into the system
Connected tooling GitHub CLI, repo, Pages — wired live, mid-conversation
Model & effort, tuned per task High effort for design calls. Fast/cheap for repetitive edits
UX/UI Pro Max ui-ux-pro-max → design-system search, before a single pixel was placed
UX writing pass ux-copy → every headline and CTA reviewed for clarity and tone
